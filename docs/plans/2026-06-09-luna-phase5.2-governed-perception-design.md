# Luna Phase 5.2 — Governed Perception Transport (design v2)

**Status:** v2 — Codex gpt-5.5 (codebase/security) + Luna (platform) reviews folded (§9); pending operator sign-off · **Date:** 2026-06-09 · **Owner:** nomade

**Directive (Simon):** capability-first toward "Luna controls any app on my desktop from chat." P5.1 (signed bounded actuation) is shipped + live-validated. P5.2 is the next rung: **Luna can SEE the screen — governed.** (PR4b productionization runs as a parallel governance track.)

**Scope discipline (Luna P5.2 spec):** screenshot **capture → redact → store(TTL) → audit**, on the **single** v2 session SSE — **TRANSPORT ONLY.** P5.2 **must NOT feed any planner / LLM / human / replay / summary / OCR / decision surface.** Perception is an unbounded, untrusted input channel; it cannot reach any consumer until the P5.3 safety contract (immutable session scope + deterministic validator + sensitive-action gate) exists. **The crux of P5.2 (both reviews): there is NO byte-retrieval path of any kind — the bytes are captured, governed, audited, stored under a hard TTL, and expired. Nothing reads them.** Consumption (with a real redactor) is P5.3/P5.4.

Builds on shipped Phase 3/4 + Phase 5.1; grounding workflow `w3d7qpepm`.

## 2. Goals / non-goals

**Goals**
- **Governed capture** (gated at the source): per `desktop_observe`/`desktop_control_enabled` capability **+** session ownership **+** secure-input fail-closed **+** target/frontmost-window scope (via a new CGWindowID resolver) **+** Luna-own-window exclusion **+** tenant isolation **+** explicit observe audit — all *before* any bytes are stored. Surfaced through the kernel (`alpha desktop observe`).
- A **redaction stage** as a pipeline placeholder; v1 policy = capture-scoping + secure-input-deny; the audit emits `redaction_status = not_planner_safe` (explicitly NOT P5.3-grade).
- **API-only quarantine transport**: redacted bytes land in an **API-private store not mounted into code-worker / CLI / any agent runtime**, under a **hard, aggressively short TTL**, with a byte-free **reference** (`resource_referenced`: `resource_id`, hash, size, `redaction_status`, `expires_at`) on the **single** v2 SSE.
- **Audit policy decisions, not pixels**: requestor/session/shell/window/app ids, capability result, secure-input result, own-window-exclusion result, byte hash/size, TTL, redaction_status, cleanup outcome — display-safe, on the single SSE.
- **Hard TTL enforced twice**: store-level `expires_at` (permanently unusable once passed, even after P5.4 exists) **+** a frequent cleanup that physically deletes.

**Non-goals (explicit — both reviews)**
- **NO byte-retrieval path of ANY kind in P5.2**: no API route, internal/MCP route, admin route, signed URL, file-browser path, support/debug/"temporary" dev endpoint, UI render. P5.2 captures, governs, stores, audits, expires — **it does not read.**
- **NO reuse of the chat screenshot upload path** (it feeds `image_b64` into the planner — `services/chat.py:503`).
- **NO observability backdoors** (Luna): the no-bytes rule also covers logs, traces, analytics, crash reports, devtools, fixture snapshots, support tooling, replay. Bytes never appear in any of these.
- **NO planner / memory / knowledge-graph / decision consumption** (P5.3/P5.4).
- No durable archival, no S3/KMS/object-store, no OCR-grade redaction (over-design for a short-lived transport buffer no one reads).
- No live video/streaming; no change to the shipped P5.1 actuation path.
- Future MCP perception MUST wrap the same `alpha desktop observe` verb + tool-group scope (Luna L-N4).

## 3. Design decisions (v2 — reviews folded)

1. **API-only quarantine store, NOT the workspace volume** (Codex B1 / Luna B). New API-private root `OBSERVATION_QUARANTINE_ROOT` (default `/var/agentprovision/observations/`), a **separate** named volume mounted **only on `api`** — never on `code-worker`, the CLI runtimes, or any agent container. Path `…/<tenant_id>/<session_id>/<artifact_id>.png`, `0700`. (Verify in CI/compose/Helm that no agent service mounts it.)
2. **No byte-retrieval endpoint exists in P5.2** (Codex B2 / Luna B). The `perception_artifacts` row + the `resource_referenced` event carry hash/size/status/`expires_at` only — **no `download_url`, no GET-bytes route.** The governed consumption path (with the real redactor + validator) is built in P5.3/P5.4. This makes "no planner/human read" true *by construction* — there is simply no code path that returns the bytes.
3. **Capture is gated at the source** (Luna B): `capture_screenshot` (governed variant) requires, fail-closed, **all** of: capability (`desktop_control_enabled` master, read+enforced in P5.2 itself — Codex I3 — converging with PR4b), session ownership, secure-input inactive (extend `secure_input_is_active()` to gate capture), a resolvable target/frontmost **CGWindowID** (new resolver — Codex I4) that is **not** Luna's own window, tenant scope. Any failure → deny + audit, **no bytes captured/stored.**
4. **Redaction = labeled placeholder** (both): v1 emits `redaction_status = not_planner_safe`; the pixel/text redactor is built in P5.3 *before* the first consumer. Acceptable now precisely because nothing reads the bytes (no LLM/human/log).
5. **Upload auth = user-JWT + `X-Device-Token`** (Codex I5), matching the claim/complete shell-device binding (`desktop_control.py:468`) — NOT generic internal-key (which would let any internal-key holder inject fake observation artifacts). Size-capped (`MAX_SCREENSHOT_SIZE` ≈ 8MB; a window PNG is small).
6. **Hard TTL, twice** (Codex I2 / Luna): store-level `expires_at` (default `PERCEPTION_ARTIFACT_TTL_MINUTES` = **15**, max 60) enforced at every access point (none in P5.2, but enforced when P5.4 adds one) **+** a **frequent** cleanup (every 5–15 min, catches up on startup, scans DB `expires_at`) — NOT the daily LearningAudio sweep. Access-TTL is a hard contract; physical deletion is frequent-best-effort and explicitly documented as such.
7. **One kernel verb**: `alpha desktop observe request|audit` → thin v1 routes delegating to the same service entrypoints. P5.2 implements `observe request` (governed capture+quarantine-store) + `observe audit` (the artifact lifecycle). `publish_session_event` from the shared service so every viewport sees identical audit.
8. **`resource_referenced` payload aligns with the renderer** (Codex N1): use `resource_id` (not `artifact_id`) + `resource_type:"screenshot"`, `hash`, `size_bytes`, `redaction_status`, `expires_at`. No `download_url`.

## 4. Architecture (capture → govern → quarantine(TTL) → byte-free reference → cleanup)

```
   alpha desktop observe request          ┌──────────── Luna Tauri client (Rust) ─────────────┐
   (human now; agent only after P5.4) ───▶│ governed_capture():                                │
                                          │  1. capability gate (desktop_control_enabled)      │
                                          │  2. session-ownership check                        │
                                          │  3. secure_input fail-closed (NEW: gate CAPTURE)    │
                                          │  4. resolve frontmost CGWindowID (NEW resolver);    │
                                          │     deny if it is com.agentprovision.luna's window  │
                                          │  5. screencapture -l <windowid>  (window-scoped)    │
                                          │  6. redaction stage (v1: scoped; status=            │
                                          │     not_planner_safe)                               │
                                          │  7. emit observe audit (requested/captured/denied)  │
                                          └───────────────┬────────────────────────────────────┘
                                                          │ multipart POST (PNG + metadata),
                                                          │ user-JWT + X-Device-Token, size-capped
                                                          ▼
   POST /api/v1/desktop-control/observations   (thin route → service entrypoint)
     • enforce desktop_control_enabled + session ownership + device-token binding (fail-closed)
     • write OBSERVATION_QUARANTINE_ROOT/<tenant>/<session>/<artifact>.png   (API-ONLY volume, 0700)
     • row perception_artifacts(artifact, tenant, session, shell, device, sha256, size,
                                redaction_status='not_planner_safe', source_window_bundle_id,
                                expires_at = now + TTL, deleted_at NULL)
     • publish_session_event(session, "resource_referenced",
          {resource_id, resource_type:"screenshot", hash, size_bytes,
           redaction_status, expires_at})         ← byte-free; NO download_url
     • return {artifact_id, expires_at}
                                                          │
                            ❌ NO GET-bytes / signed-URL / UI-render / MCP / admin / debug route
                                                          │
                                                          ▼
   PerceptionQuarantineCleanup (every 5–15 min; startup catch-up; scans DB expires_at):
     • perception_artifacts WHERE expires_at <= now AND deleted_at IS NULL
     • hard-delete file, set deleted_at, publish_session_event "resource_expired" (audit)
```

**Single-SSE adherence (Luna L-I4):** all lifecycle events (`observe_capture_requested|captured|denied`, `resource_referenced`, `resource_expired`) ride the one per-session v2 stream via `publish_session_event` (byte-free, display-safe). No second stream. **Observability-backdoor coverage:** the bytes appear in NO log/trace/metric/fixture/crash-report — only the hash/size/status do.

**New persistence (one small table):** `perception_artifacts` — `id`, `tenant_id(fk,idx)`, `session_id(fk)`, `shell_id`, `device_id`, `artifact_type`, `storage_path`, `sha256`, `size_bytes`, `redaction_status`, `source_window_bundle_id`, `expires_at(idx)`, `created_at`, `deleted_at(null)`. Bytes on the API-only disk; the row is the metadata + cleanup handle (distinct from the append-only `desktop_command_events` log).

## 5. Security / privacy model

- **No-read by construction:** there is no code path in P5.2 that returns the bytes — to a planner, a human, the UI, a log, a trace, an analytics event, a crash report, a fixture, an MCP/admin/debug route, or replay. (Both reviews' central requirement.)
- **Capture minimization at the source:** window-scoped, Luna's own window excluded, secure-input-denied, capability+ownership gated — sensitive content is largely *not captured* in the first place.
- **API-only quarantine:** bytes never touch the agent-shared workspace volume or any CLI-readable path; the store is a `0700` API-private volume verified un-mounted from agent runtimes.
- **Fail-closed everywhere:** capability OFF, secure-input active, unresolvable/own-window target, ownership/device-token mismatch, storage failure → deny + audit, nothing stored.
- **Hard TTL:** short `expires_at`, frequent physical deletion; expired artifacts permanently unusable even after P5.4 adds a consumer.
- **`not_planner_safe` labeling:** every artifact is explicitly marked un-validated until P5.3 redacts+validates it.
- **Tenant + session + device isolation** on capture + store; cross-tenant/session denied.

## 8. Risks / open items

- **CGWindowID resolver** (Codex I4): macOS API to get the frontmost app's CGWindowID; fail-closed if unresolved or if Luna is frontmost. Overlaps slice-3 live-target-window — build the window-identity reader once, shared.
- **Cleanup frequency vs Temporal/scheduler outage** (Codex I2): physical deletion is frequent-best-effort; the access-TTL (enforced when P5.4 adds a reader) is the hard guarantee. Document both. Consider an api-process-local sweeper (not only Temporal) so a worker outage can't strand bytes.
- **Volume isolation proof** (Codex B1 / Luna): a CI/compose/Helm assertion that `OBSERVATION_QUARANTINE_ROOT` is mounted on `api` only — the no-leak invariant must be test-enforced, not just intended.
- **Flag convergence**: P5.2 reads+enforces `desktop_control_enabled` itself (capability-first); ensure PR4b binds the same flag so the tracks don't drift.
- **What stores the bytes for, if nothing reads them in P5.2?** — they're the transport buffer the P5.4 governed consumer will read (after P5.3 validation). If the operator prefers, P5.2 can ship **capture+govern+audit with bytes discarded** (no quarantine store at all) and defer the store to P5.4 — *open question for sign-off* (recommendation: keep the quarantine store so P5.4 is purely additive, since no-read-by-construction already neutralizes the risk).
