# apps/embedding-service

Rust gRPC service that produces 768-dim embeddings via [fastembed](https://crates.io/crates/fastembed) (ONNX Runtime), using `nomic-ai/nomic-embed-text-v1.5`. ~2-5× faster than the Python sentence-transformers fallback in `apps/api/app/services/embedding_service.py`.

## Wiring

- Listens on **gRPC port 50051**.
- Service contract: [`proto/embedding.proto`](proto/embedding.proto). Stubs generated by `build.rs` via `tonic-build`.
- API client: `apps/api/app/services/embedding_service.py` → `embed_text()` routes here when `USE_RUST_EMBEDDING=true` and `EMBEDDING_SERVICE_URL` is set; otherwise falls back to Python sentence-transformers.
- Used by: knowledge graph entities, chat messages, memory activities, RL experience state, skill auto-trigger, email attachments, `memory-core` recall path.

## Why fastembed (not candle)

The Rust embedding service migrated from `candle` to `fastembed`. `candle` couldn't load the nomic model. `fastembed` uses ONNX Runtime under the hood and works.

## Run locally

```bash
cd apps/embedding-service
cargo run --release
```

Type check:

```bash
cargo check
```

## Required env

| Var | Default | Purpose |
|-----|---------|---------|
| `EMBEDDING_PORT` | `50051` | gRPC port |
| `EMBEDDING_MODEL` | `nomic-ai/nomic-embed-text-v1.5` | model identifier (fastembed downloads on first run) |
| `HF_HUB_DISABLE_XET` | `1` | required — HF xet protocol hangs |
| `MODEL_CACHE_DIR` | platform default | where ONNX model is cached |

## Probe / liveness

The pod has a 120s probe delay because the ONNX model is downloaded on first boot. After that, startup is fast.

## Container image

Built from `Dockerfile` (multi-stage Rust → distroless or similar). Built and deployed via CI — don't build locally.

## Memory-First context

This is the **embedding leg** of the memory-first path. The other leg is [`../memory-core/`](../memory-core/) which does the recall + record + ingest using these embeddings against pgvector. Phase 2 dual-read enabled — Python primary, Rust shadow comparison for cutover validation.
